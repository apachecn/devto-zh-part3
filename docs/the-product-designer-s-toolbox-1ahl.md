当每个问题都有无数不同的答案时，事情就变得混乱了。很快。

## 前言

木匠用他们最喜欢的锤子。开发人员使用他们喜欢的 ide。医生用他们最喜欢的…听诊器？

产品设计师用什么？

作为一个相对较新的设计师，我在为一个特定的任务寻找合适的工具时感到不知所措。当每个问题都有多个答案时，事情就变得混乱了。很快。在我冒险进入产品设计的奇妙世界之前，我的主要工具如下:

*   笔和纸

*   箱式切割机

*   InDesign

*   插图画家

当然，我使用这些工具主要是为了我的大学课程，这是面向物理工艺。我熬夜测量、裁剪、再裁剪。胶合、剥离和重新胶合。这是一个痛苦的、重复的过程，但却是值得的。不要误解我，我喜欢印刷设计，我喜欢用真实的实物来制作设计。我并不想批评那些以这种方式工作的设计师，但是我已经爱上了产品和****~数字~*** *设计。不幸的是，我的学校除了一门课程(用户界面设计…用 Photoshop)之外，没有一门课程专注于网页/移动设计😟).

因此，为了获得产品设计师的头衔，我必须完全沉浸其中，自学基础知识。正如我前面提到的，这是一项艰巨的任务，尤其是对一个新设计师来说*——*但也是一项有趣的任务。希望这篇文章能回答一些新设计师对产品设计和相关工具的疑问。

**免责声明:**本文不会有任何确定的选择。每个设计师的工作方式都不一样，我推荐的工具不一定最适合你。这篇文章仅仅是为了帮助你在你的工作流程中为正确的工作找到正确的工具。

## 头脑风暴和构思

### 纸笔

当英国作家爱德华·布尔沃-利顿在 1839 年创造了“笔比剑更有力量”这个短语时，他并不是在吹捧纸和笔相对于线框软件的优势。但是这个论点仍然适用。

纸和笔仍然在重要的方面统治着数字替代品:

*   快速原型制作:你经常会被要求在紧迫的期限内交付概念证明。如果你是为一个或两个屏幕设计，这听起来可能很容易，但是如果你是为整个网站地图或企业软件设计，会发生什么呢？笔和纸是你快速草图和注释的最好朋友。额外收获:看看[“快速成型”到底是什么？](https://blog.prototypr.io/what-the-hell-is-rapid-prototyping-fe101e446a78)作者[基顿·赫泽](https://blog.prototypr.io/@kh1211?source=post_header_lockup)。

*   * *轻松移交:* *笔记本草图可以在会议桌上传阅，或者在几秒钟内扫描并通过电子邮件发送给您的团队。合作者可以发表评论，勾画他们自己的想法，或者干脆扔掉它，告诉你重新开始。现在，你不庆幸你没有花三个小时在素描上吗？

*   激发创造力:作为一名设计师，最糟糕的感觉莫过于连续几个小时盯着电脑屏幕上的空白画布，却想不出下一个好主意。当你盯着像素时，它们也会盯着你。如果这种感觉袭来，打开你的笔记本。当你的手能够在纸上自由移动，并且你可以在纸上看到你的创造力的直接结果时，你就有动力了。你受到启发。你意识到你*是*一个有创造力的人，像素不只是盯着你看——它们在为你欢呼，等着你把你的绝妙想法从页面转移到屏幕上。

要了解更多纸和笔的原型制作的优势，请查看安德鲁·科威尔的文章。

## 用户流量和线框图

### [T2](#whimsical)异想天开

异想天开是一个流程图和线框的网络应用程序。异想天开可以让你快速建立流程并就想法进行合作。如果你的项目需要快速流程图和[线框](https://whimsical.co/wireframes/)的实时协作，异想天开可能是你最好的选择。

### [溢出](https://overflow.io/)

Overflow 于 2018 年 3 月登陆设计界，自此成为许多产品设计师最喜爱的工具。Overflow 类似于异想天开，是一个[跨平台工具，它与最流行的设计工具无缝集成和同步，允许设计师创建漂亮的用户流](https://blog.prototypr.io/meet-overflow-9b2d926b6093)。根据创作者的说法，Overflow 专注于三个主要好处:**同步**设计，**设计**美丽的用户流，**以最有效和最吸引人的方式呈现**。

## 低保真原型制作

**低保真度(lo-fi)原型制作**是一种将高级设计概念转化为有形和可测试工件的快速简单的方法。lo-fi 原型的第一个也是最重要的作用是检查和测试功能，而不是产品的视觉外观。"

### [草图](https://www.sketchapp.com/)

啊，素描。许多设计师的用户界面设计工具的卫冕冠军。看看任何“最佳 UI 设计工具列表”, Sketch 通常会排在最前面，这是有原因的。

Sketch 是一个设计工具包，旨在帮助您创作作品*——*从最早的想法到最终作品。当你第一次启动 Sketch 时，它会显得非常小。我第一次打开 Sketch 的时候，我想，“我不可能用它来设计一个完整的应用程序。”我是多么的天真。

草图已经发展成为一种动力设计工具。如今，我们可以使用[资源进行快速原型制作](https://www.sketchappsources.com/)、[数百个插件来增加功能](https://sketchapp.com/extensions/plugins/)，最棒的是，Sketch 可以兼容其他一些最好的原型制作工具，比如 [InVision](https://www.invisionapp.com/craft) 、 [Figma](https://help.figma.com/import/importing-sketch-files) 、 [Framer](https://framer.com/sketch/) 等等。

## 高保真原型制作

**高保真(hi-fi)原型**出现，其功能尽可能与即将发货的实际产品相似。当团队对他们将要构建的东西有了坚实的理解时，他们通常会创建高保真原型，并且他们需要或者用真实的用户来测试它，或者从利益相关者那里获得最终设计的批准。

### 成帧器(X)

Framer 是一个强大的设计工具，声称是“你需要为 iOS、Android 和 web 构建的唯一工具。”Framer 类似于 Sketch、Figma 和 InVision Studio，因为它允许您使用它们强大的矢量编辑器创建高保真界面。使用它们的自适应 UI 构建器、CSS 就绪元素和应用程序库，您可以在 Framer 中构建任何您能想到的界面。

Framer 用他们实现的代码原型将它提升了一个档次。Framer 运行在 CoffeeScript 上，coffee script 是一种编译成 JavaScript 的轻量级编程语言。使用 CoffeeScript，您几乎可以原型化任何东西:动画、手势，甚至实时数据请求。

[Framer X 最近发布，目前处于封闭测试阶段](https://framer.com/blog/posts/x-announcement/)。这是一个全新的产品，比 Framer 更先进，允许你使用组件构建完整的交互设计系统，全部基于 [React](https://reactjs.org/) 和 [ES6](https://tc39.github.io/ecma262/) 。

### [原理](http://principleformac.com/)

Principle 是一个专注于轻松制作动画的工具。原则的时间线使它简单说明了完美的缓解，流行，或反弹。它还让您可以自由地创建新的自定义交互，即使您正在为多个屏幕进行设计。

Principle 提供[原理镜像](https://itunes.apple.com/us/app/principle-mirror/id991911319?mt=8)，让与他人分享你的原型变得毫不费力。您还可以导出独立的 Mac 应用程序供他人查看，以便于反馈和协作。

### [草图](http://sketchapp.com)

我们已经提到了 Sketch 对于 lo-fi 原型制作的好处，但是 hi-fi 原型制作呢？嗯，素描也可以做到这一点！

### [视觉工作室](https://www.invisionapp.com/)

[InVision Studio 在某种程度上是产品设计世界的新来者](https://uxdesign.cc/invision-studio-is-out-but-are-designers-migrating-to-a-new-tool-again-102949210aeb)，但 InVision 本身已经存在了相当长的一段时间，并被全球数百万设计师和团队用于在网络上创建简单、快速的原型。

Studio 允许您在一个地方完成设计、原型制作和动画制作。InVision 植根于原型设计，Studio 将其提升到一个新的水平，具有强大的矢量编辑、灵活的图层样式和像素完美的布局。通过全面的手势和即时回放，快速原型制作变得很容易。

工作室通过加入动画更进一步。您可以使用内置的预设动画或使用时间轴和缓动编辑器自定义它们。

Studio 的另一个优势是 [InVision 的设计系统管理器](https://www.invisionapp.com/design-system-manager/)，它保证即时访问您团队的全球共享组件库，从而更容易保持一致性并管理您的大规模设计。

## 从原型到产品

### [泽普林](https://zeplin.io/)

Zeplin 是另一个为团队或编码设计师必备的应用程序。

Zeplin 是一款面向 UI 设计师和前端开发人员的应用程序，使他们能够高效协作并节省时间。设计师可以快速将他们的设计转化为强大的规范和指南，而开发人员可以在一个位置访问他们需要的所有资源，并生成针对他们工作的平台定制的代码片段。

Zeplin 与 Sketch、Figma、Adobe XD 和 Photoshop 无缝集成，可以轻松地在您最喜爱的设计工具之间进行协作。Zeplin 还支持 Slack 和 Trello 来增强您的工作流程。

### [Avocode](https://avocode.com/)

像 Zeplin 一样，Avocode 是一个独立于平台的工具，可以帮助您共享、访问和检查设计源文件。Avocode 允许您从 Sketch、Figma、Adobe XD 和 Photoshop 获得规格、资产和代码，并允许您的团队讨论变更并立即获得反馈。

Avocode 还提供了一个漂亮的版本控制系统。它为每个文件保留了无限的设计版本历史以避免冲突。再加上 Avocode 的文件管理系统，很容易跟踪更改和文件放置。

### [Hotjar](https://www.hotjar.com/)

如果这些设计工具不能帮助我们为最终用户创造更好的体验，那么它们又有什么用呢？Hotjar 是一个集分析和反馈于一体的应用程序，让你更容易真正了解你的网站和移动网站的访问者。

Hotjar 的特色是[热图](https://www.optimizely.com/optimization-glossary/heatmap/)、[录音](https://help.hotjar.com/hc/en-us/articles/115011647627-How-do-Hotjar-Recordings-Work-)、[转换漏斗](https://help.hotjar.com/hc/en-us/articles/115011656407-How-do-Hotjar-Funnels-Work-)、表单分析等等。无论您是单独工作还是在团队中工作，Hotjar 都会提供关于您的用户的必要信息，这样您就可以不断迭代和改进您的产品。

## 版本控制和文件管理

### [Dropbox](https://www.dropbox.com/h)/[Google Drive](https://www.google.com/drive/)

永远不要低估经典。云存储服务为您的产品版本控制系统提供了一个很好的起点。嵌套文件夹，与协作者共享，获得反馈，并保持迭代。

*   Dropbox: [恢复旧版本文件](https://www.dropbox.com/help/security/recover-older-versions)

*   Google Drive: [查看活动和文件版本](https://support.google.com/drive/answer/2409045?co=GENIE.Platform%3DDesktop&hl=en)

### [摘要](https://www.goabstract.com/)

Abstract 去年上市，在设计界掀起了波澜。Abstract 是一个现代设计团队合作的平台，为您的设计文件提供了一个安全的、版本控制的中心。Abstract 通过你的浏览器提供可访问性，还提供了一个 macOS 的应用程序。

Abstract 构建并扩展了 Git 的稳定技术来托管和管理您的工作。 [Abstract 滑稽的操作指南](https://www.goabstract.com/how-it-works/)展示了创建、维护、协作和更新项目是多么容易。

Abstract 目前只支持草图文件，但预计很快会有更多的文件格式。

### [植物](https://www.google.com/search?q=plantapp+version+control&source=lnms&sa=X&ved=0ahUKEwiNm52YtdfcAhUJx58KHfdqDDQQ_AUICSgA&biw=1402&bih=757&dpr=2)

工厂是设计版本控制的另一种选择。Plant 保留详细的版本历史，并提供选择性画板版本。与 Abstract 不同，Plant 提供了 Sketch *的直接插件和*独立的应用程序。Plant 可通过网络访问，这使得邀请设计师参与项目、在彼此之间同步更改以及看到项目发展变得很容易。

### [主干](https://www.jointrunk.com/)

现场最新的新人！似乎设计版本控制应用程序正变得和项目管理应用程序一样无处不在*——*但是竞争是一件好事。

Trunk 是安全版本控制平台的另一种选择，是专门为设计者设计的。Trunk 采用自动化版本控制方法，告诉您忘记“分支、拉请求和合并”术语。对于那些不熟悉版本控制的人来说，这可能是一个很好的起点。

Trunk 支持草图和 Photoshop 文件。

### 结论

为特定的工作选择合适的工具并不总是容易的，尤其是对新设计师来说。希望这篇文章能回答你可能有的关于随手可得的过多设计工具的一些问题。我的建议是:在不同的项目中广泛使用每一个，并找出哪一个适合你的工作流程。在你开始申请之前，搜索一下招聘信息，看看公司要求你了解什么，这样你就可以自信地去参加面试，准备好设计。最后但同样重要的是，玩得开心！
