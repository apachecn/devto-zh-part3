# 你应该遵循一个特定主题的教程，并将其应用到你的项目中吗？🤔

> 原文：<https://dev.to/inspiredprogrammer/should-you-follow-a-tutorial-about-a-specific-topic-and-implement-it-into-your-project-53jh>

我正在为我的项目[“server monitor”](https://monitor.inspiredprogrammer.com)开发一个认证功能。我对这样一个话题的知识相当少，所以我在 YouTube 上搜索了一个[教程来完成它。](https://youtu.be/803Ei2Sq-Zs)

[![](img/6b934f55aecb29e450d38757e8740df2.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--oiLW0PXY--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/0mzbbcsjr0pxuivgl7g1.png)

完成这个教程对我帮助很大，但是感觉不对。到目前为止，我编写的所有程序都是本教程的一对一副本，这让我想到了下面的问题。

你是否应该遵循某个特定主题的教程，并将其应用到你的项目中？

我的意思是，你做了很多事情，学到了一些东西，但这不是你自己的代码。你是否应该试着阅读文档，试着自己解决所有问题？
